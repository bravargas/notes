# Prompt 2

Focus only on implementing state persistence and rollback support for the existing PowerShell script "Invoke-ArchitectDeploymentRunner.ps1".

The script already exists. Do not rewrite the whole script.

Coding standards:
- PowerShell 5 compatible
- Use approved verbs only
- Every function must use [CmdletBinding()]
- All parameters must be explicitly typed
- Use Write-Host for logging
- Log format must be: <FunctionName>:: <message>
- Use $MyInvocation.MyCommand.Name in all functions
- Log all received parameters
- Use try/catch/finally in every function
- Do not throw strings
- Preserve original exceptions using System.Exception with InnerException
- $null comparisons must use $null on the LEFT side

Required additions:

1. Add a function named Save-DeploymentState.
It should:
- Receive WorkspacePath and State hashtable/object
- Save the state as JSON to:
  <WorkspacePath>\deploymentState.json
- Validate WorkspacePath exists
- Return the full state file path

2. Add a function named Get-DeploymentState.
It should:
- Receive WorkspacePath
- Read <WorkspacePath>\deploymentState.json
- Validate the file exists
- Convert JSON back into a PowerShell object
- Validate the following required fields exist and are not empty:
  - ArchitectServiceName
  - ArchitectServiceStatus
  - ArchitectServiceWasRunning
  - ArchitectPortalFolder
  - ArchitectDataSpace
- Return the state object

3. Add a function named Invoke-ArchitectDeployFailed.
It should:
- Receive PackageRoot, WorkspacePath, ArchitectDeployPath, ArchitectBackupPath, and OctopusMachineRoles
- Load deployment state using Get-DeploymentState
- Execute DeployFailed.ps1 from PackageRoot
- Pass these parameters to DeployFailed.ps1:
  - ArchitectDeployPath
  - ArchitectPortalFolder
  - ArchitectBackupPath
  - ArchitectServiceName
  - ArchitectServiceStatus
  - ArchitectServiceWasRunning
  - ArchitectDataSpace
  - OctopusMachineRoles
- Reuse Invoke-ExternalPowerShellScript if possible

4. Modify the catch block around Deploy.ps1 execution:
- If Deploy.ps1 fails, call Invoke-ArchitectDeployFailed
- If DeployFailed.ps1 also fails, preserve that exception but do not lose the original Deploy.ps1 exception
- Cleanup must still run from the outer finally block

5. Do not invent values for the state.
For now, add a clearly marked placeholder section showing where the deployment state should be created or imported.

Output:
- Only show the new functions and the minimal modifications needed to the existing Invoke-ArchitectDeployment function.
- Do not output the entire script.
